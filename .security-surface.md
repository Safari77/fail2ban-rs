# Security Surface

Attack surface map for fail2ban-rs. Generated 2026-03-25.

This file is the primary input for security audits. Update it after adding
listeners, parsers, endpoints, auth boundaries, or security policies.

## Sources

Log inputs consumed by the daemon. These are attacker-influenced:

- log-file-watcher (src/watcher.rs) — File I/O tailing | untrusted | none
- journal-watcher (src/watcher_journal.rs) — journalctl subprocess stdout | untrusted | none
- control-socket-listener (src/control.rs) — Unix domain socket, length-prefixed JSON | local | filesystem permissions (0o660)
- config-loader (src/config.rs) — TOML file read + config.d overlay merge | config | filesystem permissions
- maxmind-mmap (src/tracker_maxmind.rs) — Memory-mapped .mmdb file read | config | filesystem permissions

## Sinks

State changes and effects produced by the daemon:

- nftables-backend (src/executor_nftables.rs) — nft subprocess execution (add/delete elements, sets, chains) | local | resolved binary path
- iptables-backend (src/executor_iptables.rs) — iptables/ip6tables subprocess execution (chain create, rule insert/delete) | local | resolved binary path
- script-backend (src/executor_script.rs) — sh -c execution of user-defined ban/unban commands with IP/jail substitution | local | IpAddr type safety + jail name validation
- etch-wal-store (src/ban_state.rs, src/server.rs) — Persistent WAL-backed key-value store for ban records and ban counts | local | filesystem permissions
- webhook-notifier (src/webhook.rs) — curl subprocess POST to user-configured webhook URL | outbound | none (no TLS verification control)
- tell-logger (src/logging.rs) — Tell SDK client sending structured log entries to remote collector | outbound | API key (32-char hex)
- state-migration (src/server.rs) — Renames old state.bin files during format migration | local | filesystem permissions

## Transforms

Data processing between sources and sinks:

- regex-matcher (src/matcher.rs) — Two-phase Aho-Corasick + regex matching on log lines | untrusted | none
- pattern-expander (src/pattern.rs) — Expands <HOST> placeholder into IPv4/IPv6 capture regex | config | none
- ip-extractor (src/matcher.rs::extract_ip) — Parses IpAddr from regex match span via token splitting | untrusted | std IpAddr parser
- date-parser (src/date.rs) — Extracts timestamps from log lines (syslog, ISO 8601, epoch, common) | untrusted | none
- failure-tracker (src/tracker.rs) — Failure counting with circular ring buffers, ban threshold check | internal | none
- ban-calculator (src/ban_calc.rs) — Ban time escalation with multipliers, factors, and maxtime cap | internal | config validation
- ignore-list (src/ignore.rs) — CIDR allowlist + local IP self-detection via getifaddrs(2) | config | none
- duration-parser (src/duration.rs) — Parses human-friendly duration strings (10m, 1h) into seconds | config | overflow check
- config-validator (src/config.rs::validate) — Validates jail names, ports, protocols, patterns, factors | config | none

## CLI

Local command-line interface:

- cli-parser (src/main.rs) — clap-derived CLI with subcommands (run, status, ban, unban, reload, etc.) | local | none
- cli-control-client (src/control.rs::send_request) — Connects to Unix socket, sends length-prefixed JSON | local | filesystem permissions on socket
- regex-tester (src/regex_tool.rs) — Tests user-provided patterns against log lines (offline) | local | none
- dry-run (src/main.rs::dry_run) — Analyzes log file without banning (offline) | local | none
- gen-config (src/filters.rs) — Generates jail TOML from built-in templates | local | none

## Trust Boundaries

- log-to-matcher: log-file-watcher / journal-watcher -> regex-matcher — No authentication. Log lines are attacker-controlled content. Bounded line length (64 KB cap in watcher). Regex execution is the trust boundary.
- matcher-to-tracker: regex-matcher -> failure-tracker — IpAddr parsed by std, type-safe. Jail ID is internal string from config, not from log.
- tracker-to-executor: failure-tracker -> nftables-backend / iptables-backend / script-backend — IpAddr.to_string() is safe for shell context. Jail names validated at config load (alphanumeric + hyphen + underscore, max 64 chars). Ports validated as u16. Protocol validated against allowlist.
- control-socket-boundary: cli-control-client -> control-socket-listener — Unix socket with 0o660 permissions. Parent directory 0o750. 64 KB message size limit. JSON deserialization into typed Request enum (no arbitrary code paths).
- config-to-daemon: config-loader -> all modules — TOML parsed into typed structs with validation. Config file permissions are operator responsibility.
- maxmind-mmap-boundary: maxmind-mmap -> ip-extractor — unsafe mmap. File must not be modified while mapped. World-writable check warns but does not block.
- webhook-outbound: webhook-notifier -> external HTTP endpoint — curl subprocess. No TLS certificate pinning. URL from config. Payload contains IP addresses and jail names.
- tell-outbound: tell-logger -> external collector — API key in config file (plaintext). Payload contains IP addresses, jail names, ban metadata.
- script-backend-boundary: config-loader -> script-backend — User-defined shell commands executed via sh -c. <IP> substitution uses IpAddr (safe). <JAIL> substitution uses validated jail name (safe). The command template itself is trusted config.
- install-script-boundary: GitHub Releases -> customer-infra — HTTPS download with sha256 checksum verification. FAIL2BAN_SKIP_CHECKSUM can bypass.

## Unsafe Code

One instance:

- src/tracker_maxmind.rs:133 — `unsafe { maxminddb::Reader::open_mmap(path) }` — Memory-maps a MaxMind .mmdb file. Safety invariant: file must not be modified, truncated, or replaced non-atomically while the mmap exists. World-writable permission check is present but advisory only.

## Cross-Cutting Patterns

- Process execution: The daemon spawns external processes for firewall management (nft, iptables, ip6tables), webhook delivery (curl), and journal reading (journalctl). Binary paths for firewall tools are resolved at startup from fixed system directories (/usr/sbin, /sbin, /usr/bin, /bin), preventing PATH injection.
- Config reload: SIGHUP triggers hot-reload. New config is parsed and validated before old watchers are cancelled. Tracker jail params are updated atomically via channel message. No partial-reload state.
- Shutdown ordering: SIGINT triggers graceful shutdown. Firewall teardown runs before cancellation. Etch store is flushed. Tell client is closed. 500ms drain delay for task completion.
- State persistence: Ban state is WAL-persisted via etchdb. Expired bans are purged on startup. Ban counts survive restarts for escalation continuity.
- Channel backpressure: Failure and executor channels are bounded (configurable, default 1024). Control channel is bounded at 32. Backpressure prevents unbounded memory growth under load.
